## Q&A Format: Manual Comment Integration Logic

This guide explains how to intelligently integrate manual comments into the Q&A review format.

### Core Principle:
Manual comments must EITHER be integrated into existing Q&A pairs OR form new Q&A pairs. Never leave comments unaddressed.

### Step-by-Step Process:

**STEP 1: Analyze Each Manual Comment**
For each manual comment provided, identify:
- What feature/topic is it discussing?
- What specific information does it provide?
- Is this information critical to include?

**STEP 2: Check for Existing Question Match**
Review all the pre-defined questions in the section template and ask:
- Does any existing question already cover this feature/topic?
- Would this comment naturally fit as additional information in an existing answer?

Examples of matches:
- Comment about "VPN works but slow" → Matches existing Q: "Can I use a VPN to play in [casino]?"
- Comment about "KYC took 3 days for me" → Matches existing Q: "How long can KYC reviews take?"
- Comment about "They have 15 exclusive slots" → Matches existing Q: "Does [casino] have Originals or provably fair games?"

**STEP 3a: If Match Found - Integrate into Existing Answer**
- Add the comment information to the relevant existing question's answer
- Maintain the answer's existing structure and flow
- Incorporate the new information naturally without disrupting the answer
- Keep the original answer content and ADD the new information from the comment
- Make sure the added information is clearly stated with all details from the comment

Example:
Existing Q&A:
Q: Can I use a VPN to play in [casino]?
A: Yes, [casino] is VPN friendly, which means you can access it from restricted regions...

After adding comment "VPN works but connection drops sometimes":
Q: Can I use a VPN to play in [casino]?
A: Yes, [casino] is VPN friendly, which means you can access it from restricted regions. However, I noticed that VPN connections can drop occasionally, which might interrupt your gaming session.

**STEP 3b: If No Match Found - Check for Consolidation Before Creating New Q&A**

CONSOLIDATION RULE: Before creating a new Q&A, ask yourself:
- Would this new Q&A have fewer than 4 sentences?
- Does it partially overlap with information in an existing answer (e.g., KYC info that's already in the restrictions answer, deposit details that expand on the altcoin answer)?
- Could this information make an existing answer more complete rather than standing on its own?
- Does it restate or expand on a detail already mentioned in another answer?

If YES to any of these: MERGE the comment into the most relevant existing answer instead of creating a thin standalone Q&A. A short paragraph added to an existing comprehensive answer is better than a separate 2-sentence Q&A that fragments information.

EXAMPLES OF WHAT SHOULD NOT BECOME STANDALONE Q&As:
- "Are there fiat deposit options?" → merge into the altcoin/deposit answer
- "What happens if I have multiple pending withdrawals?" → merge into the withdrawal restrictions answer
- "Are the wagering terms clearly explained?" → this is filler, not a real question. Skip it or weave the observation into the bonus terms answer.
- "Do I need to complete verification to deposit?" → merge into the withdrawal restrictions or anonymity answer
- Any Q&A that would essentially repeat information already present in another answer with slightly different wording

Only create a NEW Q&A pair when the comment describes a topic that is genuinely UNRELATED to any existing question AND has enough substance to warrant its own section (at least 4-5 sentences of useful, specific content).

When creating a new Q&A:
- Formulate a BROAD, GENERAL question that a player would naturally ask WITHOUT knowing the specific feature exists. The comment reveals the answer, but the question should be something a curious player would search for.
- WRONG approach: comment about a blockchain token → "Does [casino] have a blockchain token?" (too specific, assumes knowledge)
- RIGHT approach: comment about a blockchain token → "Are there any unique reward systems at [casino]?" or "What special features does [casino] offer?" (broad, discoverable)
- Write a complete answer using all information from the comment
- Follow the same question format as other questions in that section
- Place the new Q&A pair under the "+ questions/answers from manual comments" section

Question Formulation Guidelines:
- Write questions as a real player would type them into a search engine BEFORE knowing about the feature. The player doesn't know the feature exists yet, so the question should be broad enough that someone would naturally ask it.
- BAD: "Does TrustDice have their own blockchain token?" -- too specific, no player would ask this without already knowing it exists
- BAD: "What is the nature of the loyalty program structure?" -- too formal
- BAD: "How does the Rain feature work in Bombastic's chat?" -- assumes knowledge of a specific feature
- GOOD: "Are there any unique features at TrustDice?" -- broad, natural, leads to the specific answer
- GOOD: "Does [casino] have a cashback or loyalty program?" -- natural
- GOOD: "Can I withdraw without completing KYC?" -- direct, practical
- GOOD: "What community features does Bombastic offer?" -- broad enough to discover the Rain feature through the answer
- The ANSWER is where you get specific. The QUESTION should be what a curious player would search for.
- Use natural phrasing: "Can I...", "Is there...", "How does...", "What happens if...", "Are there any...", "What makes..."
- Avoid yes/no questions unless they lead to detailed answers

Example comment → Q&A conversion:

Comment: "The casino offers a unique cashback program that gives 10% daily cashback on losses, no wagering required."

New Q&A:
Q: Are there any standout promotions at [casino]?
A: Yes, [casino] offers a daily cashback program that returns 10% of your losses with no wagering requirements. You get real money back without any playthrough conditions.

Comment: "The casino has its own blockchain token called XYZ on the EOS chain, earnable through play-to-earn."

New Q&A:
Q: What unique features does [casino] offer?
A: [Casino] has its own blockchain token called XYZ, issued on the EOS chain. You can earn it through a play-to-earn system where...

(Note: the question is broad because a player wouldn't know about the token beforehand. The answer then introduces the specific feature.)

**STEP 4: Categorization Rules**
Place new Q&A pairs in the appropriate section based on topic:
- **General**: Overall experience, VPN, reputation, support languages, website issues, regional restrictions, UX, standout features
- **Payments**: Deposit/withdrawal methods, processing times, limits, fees, KYC, payment issues
- **Games**: Game selection, specific games, game features, providers, performance issues
- **Responsible Gambling**: Limits, tools, self-exclusion, account controls
- **Bonuses**: Any promotional offers, bonus terms, wagering, restrictions, loyalty programs, VIP programs, rakeback, cashback, reward systems, gamification features, token systems, ongoing promotions for returning players. NOTE: If a comment is about loyalty, rewards, VIP, rakeback, or any system that gives players ongoing value, it belongs in Bonuses (specifically under the ongoing promotions question), NOT in General.

**STEP 5: Quality Checks**
Before finalizing, verify:
- ✓ All manual comments have been addressed (either integrated or converted to new Q&A)
- ✓ No duplicate information across different Q&A pairs
- ✓ New questions follow the same format and style as existing questions
- ✓ All specific details from comments are included (numbers, names, exact mechanisms)
- ✓ Answers maintain the conversational, analytical tone consistent with the guidelines

### Special Cases:

**Multiple Comments About Same Topic:**
If you have multiple comments about the same feature:
- Combine them into one comprehensive answer
- OR add them as separate points within the same answer
- Do NOT create multiple Q&A pairs for the same topic

**Conflicting Comments:**
If a comment conflicts with existing data:
- Prioritize the manual comment (it's more recent/specific)
- Mention both perspectives if appropriate
- Example: "While the casino states X, in practice I found Y..."

**Vague Comments:**
If a comment is unclear:
- Do your best to interpret the intent
- Focus on the factual information provided
- If you cannot create a meaningful Q&A, integrate it into the most relevant existing answer

### Format for Manual Comment Section:

At the end of each section template, you'll see:
```
## + questions/answers from manual comments
* Include any additional [section] questions and answers provided in the manual comments section.
```

This is where you should place NEW Q&A pairs created from comments that don't match existing questions.

Format:
```
## Q: [Your formulated question]?

[Your complete answer based on the comment information]

## Q: [Next formulated question if applicable]?

[Next answer]
```

### CRITICAL REMINDERS:
- NEVER summarize or condense comment information - include ALL details
- NEVER skip manual comments - they MUST all be addressed
- ALWAYS maintain first-person perspective ("I") and address reader as "you"
- ALWAYS bold key features mentioned in comments
- ALWAYS include specific numbers, names, and mechanisms from comments
- NEVER add information not in the comment or existing data
- NEVER generate a Q&A that simply restates what the comment said without adding context or analysis -- always frame the answer in terms of how it affects the reader
